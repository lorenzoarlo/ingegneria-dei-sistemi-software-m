SistemaSJavalin (da funzione a servizio eval)

Introduction

Realizzazione (Progetto SistemaSJavalin ) di un servizio in Java che fa il calcolo dei valori di una espressione matematica.

Requirements

  • realizzare un semplicissimo servizio in Java che calcola i valori di una data espressione matematica
  • il servizio deve essere raggiungibile sia via WebSocket che via HTTP/1 e HTTP/2.
  • l'interazione deve avvenire tramite una pagina html che permetta di indicare la x per cui si vuole valutare la funzione e la modalità di interazione (WebSocket, Get o Post)
  • il deployment dell'applicazione avviene mediante Docker

Requirement analysis

  • La funzione da realizzare è eval(x) = sin(x)+cos(sqrt(3)*x), che prende in input un numero reale e restituisce un numero reale.

Si evidenzia una struttura a livelli:
  • Livello di Dominio: La funzione eval è il layer che definisce la business logic del sistema
  • Livello Infrastruttura: qui opera Javalin, con l'obiettivo di gestire i protocolli e di prelevare dai messaggi le informazioni rilevanti per il domain-layer.
  • Livello di Adattamento: livello intermedio che trasforma i dati di basso livello forniti dalla infrastruttura nei dati richiesti dalla applicazione
